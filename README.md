J'adore — это полноценное веб-приложение для кафе-бара, разработанное на стеке Java Spring Boot. Система позволяет управлять меню напитков, обрабатывать заказы клиентов и предоставляет интерактивный интерфейс с "умным" чат-ботом для подбора напитков по настроению.

🚀 Основные возможности
Управление меню (CRUD): Полный цикл управления напитками (создание, чтение, обновление, удаление).

Система заказов: Оформление заказов с привязкой нескольких напитков и расчетом итоговой суммы.

Категоризация: Группировка напитков по типам (Лимонады, Кофе, Смузи и т.д.).

Умные рекомендации: Сервис подбора напитка на основе текстового ввода пользователя (настроение/вкус).

Автозаполнение: При запуске система автоматически наполняет базу данных 25 топовыми позициями меню.

Интерактивный UI: Современный фронтенд с использованием Fetch API для взаимодействия с бэкендом.

🏗 Архитектура проекта
Проект построен по принципу Layered Architecture (Многослойная архитектура):

Presentation Layer (Controllers): Обработка REST-запросов.

Business Layer (Services): Логика обработки заказов и рекомендаций.

Data Access Layer (Repositories): Взаимодействие с базой данных через Spring Data JPA.

Database Layer: Хранение данных (H2/PostgreSQL).

📊 Проектирование системы
UML-диаграмма классов

Описывает структуру сущностей и их взаимосвязи (One-to-Many, Many-to-One).

Схема базы данных (ER-диаграмма)

База данных спроектирована в третьей нормальной форме для обеспечения целостности данных.

🛠 Технологический стек
Backend: Java 17, Spring Boot 3.x, Spring Data JPA.

Database: H2 Database (In-Memory для разработки) / Hibernate.

Frontend: HTML5, CSS3 (Custom Variables), JavaScript (Vanilla JS, Fetch API).

Build Tool: Maven.

🛣 API Эндпоинты
Напитки (Drinks)

GET /api/drinks — Получить весь список напитков.

GET /api/drinks/{id} — Получить информацию о конкретном напитке.

POST /api/drinks — Добавить новый напиток.

PUT /api/drinks/{id} — Обновить данные напитка.

DELETE /api/drinks/{id} — Удалить напиток.

Заказы (Orders)

GET /api/orders — Список всех заказов.

POST /api/orders — Создать новый заказ.
